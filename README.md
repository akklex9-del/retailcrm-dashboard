# retailcrm-dashboard

Проект: интеграция `RetailCRM -> Supabase`, дашборд заказов и Telegram-уведомления для крупных заказов.

## Что реализовано

- Импорт 50 тестовых заказов из `mock_orders.json` в RetailCRM через API.
- Скрипт синхронизации заказов из RetailCRM в Supabase.
- Веб-дашборд (Next.js) с графиком заказов/выручки из Supabase.
- Telegram-уведомления при заказе свыше `50 000 ₸`.
- Запуск в проде на VPS (PM2 + Nginx), cron каждые 5 минут.

## Какие промпты/запросы использовались

Ниже кратко по реальным запросам в процессе:

- "есть mock_orders.json — 50 тестовых заказов. Загрузи их в RetailCRM через API"
- "Напиши скрипт RetailCRM -> Supabase"
- "Сделай веб-страницу с графиком заказов"

## Где застрял и как решил

### 1) Импорт в RetailCRM: `fetch failed`

**Проблема:** импорт падал на каждом заказе.  
**Причина:** в окружении использовались шаблонные значения `.env` (`yourstore...`, `replace_me`).  
**Решение:** поправлены реальные значения в `.env` и повторный импорт.

### 2) Ошибка RetailCRM: `orderType ... does not exist`

**Проблема:** API возвращал ошибку по `orderType=eshop-individual`.  
**Причина:** в конкретном аккаунте RetailCRM нет такого кода типа заказа.  
**Решение:** в скрипте импорта убрана передача `orderType` (для тестовой загрузки это допустимо), после чего импорт прошёл успешно (`50/50`).

### 3) Дашборд показывал "старые" даты

**Проблема:** на графике отображались даты, которых нет в текущей выборке БД.  
**Причина:** артефакты оси/перерисовки графика и форматирование даты в окружении.  
**Решение:** зафиксирована категориальная ось, добавлен принудительный ремаунт графика при изменении данных, нормализовано вычисление дня (Almaty, стабильный формат `YYYY-MM-DD`).

### 4) На VPS открывался `Welcome to nginx`

**Проблема:** вместо приложения показывалась дефолтная nginx-страница.  
**Причина:** HTTPS на IP попадал в default SSL vhost.  
**Решение:** настроен отдельный default SSL server block с редиректом на рабочий HTTP endpoint приложения, проверено что дашборд открывается.

## Команды запуска

```bash
npm install
npm run import:mock
npm run sync:orders
npm run dev
```

## Прод (VPS)

- Приложение: `pm2` (процесс `retailcrm-dashboard`)
- Reverse proxy: `nginx`
- Cron:

```cron
*/5 * * * * curl -fsS http://127.0.0.1:3000/api/cron/high-value-alert >> /var/log/retailcrm-high-value-alert.log 2>&1
```
